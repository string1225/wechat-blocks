# 微信小游戏产品文档：和AI一起消方块

## 1. 项目概述

### 1.1 游戏类型
- **类型**：3D 解谜 / 点击消除
- **平台**：微信小游戏（优先），后续可扩展到 App
- **目标用户**：休闲玩家、碎片时间游戏人群（类似《开心消消乐》《我的安吉拉》用户）
- **画风**：清新简约 / 低多边形（Low Poly）风格

### 1.2 核心体验一句话描述
"点击消除 3D 方块，看着它们崩塌消失，享受解压的快感。"

## 2. 核心玩法

### 2.1 基础规则
1. **初始状态**：屏幕中央显示一个由 N×N×N 个小立方体组成的大立方体（默认 3×3×3 或 4×4×4）
2. **操作方式**：
   - **点击/触摸**一个方块 → 该方块消除
   - **滑动旋转**视角（可 360° 查看立方体）
   - **双指缩放**调整视距
3. **消除逻辑**：
   - 被点击的方块直接消除
   - 如果消除后导致上方方块**悬空** → 触发**物理下落**
   - 如果多个同色方块相邻 → 可触发**连消**（可选机制）
4. **过关条件**：
   - 模式 A：在限定步数内消除所有方块
   - 模式 B：消除到剩余方块数 ≤ 阈值
   - 模式 C：限时模式，尽可能多消

### 2.2 操作道具（底部按钮栏）
| 道具 | 功能 | 冷却/次数限制 |
|------|------|--------------|
| 🔄 重置 | 重置当前关卡到初始状态 | 无限次 |
| ↩️ 后退 | 撤销上一步操作 | 每关 5 次 |
| 💣 炸弹 | 随机消除一个方块 | 每关 3 次 |

### 2.3 难度控制
- **唯一难度变量**：矩阵大小（N×N×N）
- 关卡越高 → 矩阵越大
- 示例：关卡 1-10 为 2×2×2，11-20 为 3×3×3，依此类推
- **所有方块数据随机生成**（无预设关卡）

### 2.4 有解性保证
生成方块时必须验证：
1. 使用 BFS/DFS 模拟消除过程
2. 确保存在至少一条消除所有方块的路径
3. 如果无解 → 重新生成

### 2.5 死局检测
- 实时检测当前状态是否仍有可消除的方块
- 如果无解 → 提示用户"已锁死"，游戏暂停
- 提供：重置 / 后退 / 炸弹 选项

### 2.6 自动运行模式（挂机通关）
- 玩家可开启"自动运行"
- 系统按最优路径逐个消除方块
- 动画流畅，方块一个个抽走
- 玩家可以放置挂机观看通关过程

#### 2.2.3 物理系统
- 方块消除后，检测上方方块的支撑关系
- 悬空方块受重力影响自然下落
- 下落过程中可能触发新的悬空检测（连锁反应）
- 下落动画要流畅、有弹性感（squash & stretch）

## 3. 关卡设计

### 3.1 关卡结构
- **总关卡数**：100+（首批上线）
- **章节划分**：每 10 关一章，每章有主题和配色
- **难度曲线**：

| 章节 | 关卡 | 立方体大小 | 特殊机制引入 |
|------|------|-----------|-------------|
| 第1章 | 1-10 | 2×2~3×3 | 基础点击消除 |
| 第2章 | 11-20 | 3×3~4×4 | 引入方向箭头方块 |
| 第3章 | 21-30 | 4×4 | 引入冰冻方块 |
| 第4章 | 31-40 | 4×4~5×5 | 引入炸弹+连消 |
| 第5章 | 41-50 | 5×5 | 综合关卡 |
| 第6章+ | 51-100+ | 5×5~6×6 | 挑战模式 |

## 4. 游戏模式

### 4.1 闯关模式（主线）
- 进入游戏 → 选择"闯关" → 加载历史进度
- 按顺序通关，解锁下一关
- 关卡只控制矩阵大小，方块数据随机生成
- 每关 1-3 星评价（基于步数）

### 4.2 每日挑战
- 每天生成一个固定种子的随机关卡
- 全服玩家挑战同一关卡
- 排行榜（最少步数 / 最快时间）

## 6. 技术方案

### 6.1 技术栈选择

| 层面 | 推荐方案 | 备选方案 |
|------|---------|---------|
| **游戏引擎** | Three.js (WebGL) | Babylon.js / Cocos Creator |
| **物理引擎** | Cannon-es / Ammo.js | 自实现简化物理 |
| **框架** | 微信小游戏原生 API | Phaser + 小游戏适配层 |
| **语言** | TypeScript | JavaScript (ES6+) |
| **构建工具** | webpack / rollup | Vite |
| **资源管理** | 自建资源加载器 | Loader.js |

### 6.2 为什么推荐 Three.js？
- 微信小游戏原生支持 WebGL
- Three.js 成熟稳定，社区资源丰富
- 3D 方块渲染性能好（即使 6×6×6 = 216 个方块也毫无压力）
- 物理引擎集成方便（cannon-es）

### 6.4 性能优化要点
- **对象池**：方块消除/创建复用对象，避免 GC
- **实例化渲染**（InstancedMesh）：大量相同几何体的方块用 InstancedMesh 一次绘制
- **LOD**：远距离方块降低面数
- **纹理图集**：合并为一张 TextureAtlas，减少 draw call
- **物理简化**：不需要完整物理引擎，自定义简化的重力+碰撞即可

### 6.5 微信小游戏适配注意事项
- 不能使用 DOM/BOM API（如 `document`, `window`）
- 需要用 `wx.createCanvas()` 和 `canvas.getContext('webgl')`
- 音频用 `wx.createInnerAudioContext()`
- 本地存储用 `wx.setStorageSync()`
- 广告接入：`wx.createRewardedVideoAd()`（激励视频）
- 分享用 `wx.shareAppMessage()`
- 包体积限制：首包 < 4MB，分包每个 < 2MB

## 附录 B: 关键算法（随机生成 + 有解性验证）

### 随机生成算法伪代码
```typescript
function generateLevel(gridSize: number): Block[][][] {
  while (true) {
    const blocks = randomFill(gridSize);
    if (isSolvable(blocks)) {
      return blocks;
    }
    // 无解则重新生成
  }
}

function isSolvable(blocks: Block[][][]): boolean {
  // 模拟消除：BFS 探索所有可能的点击序列
  // 如果存在一条路径能消除所有方块，返回 true
  const queue = [{ blocks: copy(blocks), steps: [] }];
  const visited = new Set<string>();
  
  while (queue.length > 0) {
    const state = queue.shift();
    const hash = hashBlocks(state.blocks);
    if (visited.has(hash)) continue;
    visited.add(hash);
    
    if (isEmpty(state.blocks)) return true;
    
    // 尝试点击每个可点击的方块
    for (const block of getClickableBlocks(state.blocks)) {
      const newBlocks = eliminate(state.blocks, block);
      queue.push({ blocks: newBlocks, steps: [...state.steps, block] });
    }
  }
  return false; // 无解
}
```

### 死局检测（实时）
```typescript
function isDeadlock(blocks: Block[][][]): boolean {
  // 同 isSolvable，但只检查当前状态是否有至少一个可点击方块
  return getClickableBlocks(blocks).length === 0;
}
```
